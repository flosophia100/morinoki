# ワードマップエディター デバッグガイド

## 動作確認手順

### 1. 基本的な動作確認

1. **初期化の確認**
   - ブラウザのコンソールを開く（F12）
   - 以下のメッセージが表示されることを確認：
     - "ワードマップエディター（モジュール版）初期化開始"
     - "D3WordMapEditor初期化完了"
     - "UIモジュール初期化完了"
     - "IOモジュール初期化完了"
     - "ワードマップエディター（モジュール版）初期化成功"

2. **サンプルデータの表示**
   - 4つのノード（アイデア、計画、実行、評価）が表示される
   - 3つのリンクが表示される

### 2. モード切り替えテスト

1. **選択モード（デフォルト）**
   - ノードをクリック → プロパティパネルに情報表示
   - リンクをクリック → プロパティパネルに情報表示
   - 背景をクリック → 選択解除

2. **作成モード**
   - ツールバーの「作成」ボタンをクリック（または「C」キー）
   - 背景をダブルクリック → 新しいノードが作成される

3. **接続モード**
   - ツールバーの「接続」ボタンをクリック（または「L」キー）
   - 最初のノードをクリック → パルスアニメーション開始
   - 2番目のノードをクリック → リンクが作成される

### 3. リンク編集機能のテスト

1. **リンクの選択**
   - リンクをクリックして選択

2. **リンクプロパティの変更**
   - 色の変更 → カラーパレットから選択
   - 太さの変更 → スライダーで調整
   - 矢印方向の変更：
     - 「→」ボタン：一方向
     - 「↔」ボタン：双方向
     - 「—」ボタン：矢印なし
   - 線スタイルの変更：
     - 「━」ボタン：実線
     - 「┅」ボタン：破線
     - 「┈」ボタン：点線

3. **変更の反映確認**
   - 各プロパティを変更後、即座にリンクの表示が更新されることを確認

### 4. デバッグパネルの使用

1. **デバッグパネルを開く**
   - ヘッダーの「🔧 デバッグ」ボタンをクリック

2. **リアルタイムタブ**
   - システム状態とD3.js状態を確認

3. **問題検出タブ**
   - 「リンク編集テスト実行」ボタンをクリック
   - テスト結果を確認

### 5. エラーチェック項目

- [ ] コンソールにエラーが表示されていない
- [ ] すべてのモジュールが正常に読み込まれている
- [ ] ノードのドラッグ&ドロップが機能する
- [ ] リンクの編集が即座に反映される
- [ ] ファイルの保存・読み込みが機能する
- [ ] キーボードショートカットが機能する

### 6. パフォーマンステスト

1. **大量ノード作成**
   - 作成モードで50個以上のノードを作成
   - 動作が極端に遅くならないことを確認

2. **大量リンク作成**
   - 複数のノード間に多数のリンクを作成
   - レンダリングパフォーマンスを確認

## トラブルシューティング

### 問題: リンクの編集が反映されない

**確認事項:**
1. `forceRecreateLink`メソッドが呼ばれているか
2. D3.jsのデータバインディングが正しく動作しているか
3. CSS変数が正しく適用されているか

**解決方法:**
```javascript
// コンソールで以下を実行
window.wordMapEditor.debugModule.checkLinkEditingReflection()
```

### 問題: ノードが作成できない

**確認事項:**
1. 作成モードになっているか
2. ダブルクリックイベントが正しく処理されているか
3. 座標変換が正しいか

### 問題: モジュールが読み込まれない

**確認事項:**
1. スクリプトタグの順序が正しいか
2. ファイルパスが正しいか
3. JavaScriptエラーが発生していないか

## デバッグコマンド

コンソールで使用できる便利なコマンド：

```javascript
// エディターの状態確認
window.wordMapEditor.state

// データ確認
window.wordMapEditor.data

// 強制レンダリング
window.wordMapEditor.render()

// デバッグ情報取得
window.wordMapEditor.debugModule.exportDebugInfo()
```
